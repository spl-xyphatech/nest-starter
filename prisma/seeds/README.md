# Database Seeds

This directory contains seed files for populating the database with initial data.

## Files

- **categories.ts** - Seeds 15 restaurant categories (Italian, Asian, Fast Food, etc.)
- **tags.ts** - Seeds 20 restaurant tags (WiFi, Pet Friendly, Vegetarian, etc.)
- **users.ts** - Seeds 40+ merchant users with hashed passwords
- **merchants.ts** - Seeds 40+ merchants with random categories and tags
- **seed.ts** - Main seed orchestrator that runs all seed functions

## Running Seeds

### Development

Run all seeds:

```bash
npm run db:seed
```

Run migrations + seeds:

```bash
npm run db:migrate
```

Reset database and run seeds:

```bash
npm run db:reset
```

### Production

Run migrations (without seeding):

```bash
npm run db:migrate:prod
```

## Data Summary

After running the seed script, your database will contain:

- **15 Categories** - Restaurant categories like Italian, Asian, Fine Dining, etc.
- **20 Tags** - Feature tags like WiFi, Smoking Area, Vegetarian Options, etc.
- **40+ Users** - Test merchant accounts with email/password
- **40+ Merchants** - Restaurant/food service merchants with:
  - Random assigned categories (1-3 per merchant)
  - Random assigned tags (2-5 per merchant)
  - Random statuses (PENDING, APPROVED, REJECTED)
  - Generated addresses and descriptions

## Customization

To modify seed data, edit the respective TS files:

1. Update merchant names in `users.ts`
2. Update category definitions in `categories.ts`
3. Update tag definitions in `tags.ts`
4. Adjust merchant count in `merchants.ts`

## Notes

- Seeds use `upsert` to safely re-run without duplicates
- Relations (MerchantCategory, MerchantTag) are created with random assignments
- All timestamps are automatically generated by Prisma
- Deleted records are soft-deleted (deletedAt is null)

## Troubleshooting

If you encounter migration issues:

```bash
npm run db:reset  # Reset everything and reseed
```

This will drop and recreate the database, apply all migrations, and run the seed script.
